# ordenacao
<h1> Algoritmos de ordenação</h2>
<p> Implementar e testar os principais algoritmos de
ordenação, fazer análises assintóticas</p>
<ul>
  <li>Bubble sort</li>
  <li>Insertion sort</li>
  <li>Merge sort</li>
  <li>Quick sort </li>
  <li>Heapsort </li>
  <li>Radix</li>
</ul>
<p>Cada algoritmo será testado com uma quantidade de entradas aleatórias, dividindo-se em:</p>
<ul> 
  <li>10 entradas</li>
  <li>1000 entradas</li>
  <li>100.000 entradas</li>
  <li>10.000.000 de entradas</li>
<p>De forma a mensurar o tempo de forma mais precisa, os algoritmos deverão ser executados
3 vezes para cada tamanho de entrada. Assim, o tempo de execução consistirá na média
  desses tempos</p>
